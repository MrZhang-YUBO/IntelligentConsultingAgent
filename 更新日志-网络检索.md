# 更新日志 - 网络检索功能

> 版本：1.2.1 → 1.3.0  
> 日期：2026-06-24

---

## 一、功能概述

本次更新新增了**网络检索**功能，作为知识库检索的补充。当知识库中没有足够相关内容时，系统可以自动或手动触发网络搜索，将网络检索结果经安全过滤和摘要压缩后，与知识库结果合并呈现给用户。

### 核心能力

| 能力 | 说明 |
|------|------|
| **自动触发** | 混合检索最高得分 < 阈值（默认 0.3）时，自动触发网络检索补充信息 |
| **手动触发** | 前端"网络搜索"开关，用户主动启用后 Agent 同时检索知识库和网络 |
| **摘要压缩** | 网络检索结果用 qwen-turbo 压缩至 200~300 字，避免长文撑爆上下文 |
| **安全过滤** | 双层防护：规则引擎（关键词黑名单+域名屏蔽+XSS清洗）+ LLM 审核 |
| **意图识别** | 新增 `web_search` 意图类型，Agent 可识别用户"搜一下"类需求 |

## 二、数据流

### 自动触发路径（用户无需操作）

```
用户提问
  → Agent 调用 retrieve_knowledge(query)
    → 混合检索 (向量 + BM25)，结果含 _hybrid_score
    → 重排 (Rerank)，结果含 _rerank_score
    → 检查最高分 < 0.3 ?
      → 是: 调用 web_search_service.search(query)
        → Tavily API 搜索
        → 安全过滤（规则引擎 → LLM 审核）
        → 摘要压缩
        → 合并到知识库结果
      → 否: 仅返回知识库结果
    → 格式化返回给 Agent
```

### 手动触发路径（用户打开前端开关）

```
用户点击"网络搜索"开关 ON → 发送请求 {EnableWebSearch: true}
  → 系统提示词追加："请同时使用 web_search 工具搜索网络信息"
  → Agent 自主调用两个工具:
    → retrieve_knowledge(query)  [知识库结果 + 可能的自动触发]
    → web_search(query)          [网络结果（含安全过滤+摘要）]
  → Agent 综合两方面结果生成回答
  → SSE 流式返回
```

## 三、新增文件

### 1. `app/services/content_safety_service.py` — 内容安全过滤服务

**双层架构**：

- **第一层：规则引擎**（同步，快速，无 LLM 依赖）
  - 关键词黑名单过滤（大小写不敏感，配置 `web_search_safety_keyword_blacklist`）
  - 屏蔽域名检测（配置 `web_search_safety_blocked_domains`）
  - XSS/注入内容清洗：移除 `<script>` 标签、`javascript:` URL、`on*` 事件处理器

- **第二层：LLM 审核**（异步，可选，配置 `web_search_llm_review_enabled`）
  - 使用 qwen-turbo 对每篇文档做安全/合规性审查
  - 结构化输出（Pydantic）：每篇标记 `is_safe: bool` + `reason: str`
  - 批量处理（每次 LLM 调用审核最多 5 篇文档）
  - **失败时 fail-open**（放行而非误杀），规则引擎提供硬安全底线

**全局单例**：`content_safety_service`

### 2. `app/services/web_search_service.py` — 网络检索服务

**核心流程**：`Tavily API → 安全过滤 → 摘要压缩 → 返回 Document 列表`

- `search(query)` — 同步接口，供 `@tool` 调用
- `search_async(query)` — 异步接口，供异步上下文调用
- `_summarize_results()` — 每条结果独立压缩至 200~300 字，原始内容存入 `metadata._original_content`
- 延迟初始化 Tavily 客户端和摘要模型，避免启动时崩溃
- **失败时返回空列表，绝不中断主对话**

**Document metadata 字段**：

| 字段 | 说明 |
|------|------|
| `_source` | `"web_search"`（标识来源） |
| `_web_url` | 结果 URL |
| `_web_title` | 结果标题 |
| `_web_score` | Tavily 相关度分数 |
| `_result_source` | `"web_search"`（与知识库结果统一标识） |

**全局单例**：`web_search_service`

### 3. `app/tools/web_search_tool.py` — LangChain 网络检索工具

遵循 `knowledge_tool.py` 的完全相同模式：
- `@tool(response_format="content_and_artifact")` 装饰器
- 返回 `Tuple[str, List[Document]]`（格式化文本 + 原始文档）
- 功能未启用时返回"网络检索功能未启用"

### 4. `app/services/hybrid_search_service.py` — 保留融合分数

**改了什么**：`search()` 方法在返回文档前，将融合分数写入 `doc.metadata["_hybrid_score"]`。

**为什么改**：原来融合分数被丢弃（`[doc for doc, _ in scored]`），自动触发网络检索需要知道最高得分来判断是否低于阈值。

**影响**：无破坏性。下游代码只读取 `metadata.get("_file_name")` 等特定 key，新增 `_hybrid_score` 不影响。与 `rerank_service.py` 存储 `_rerank_score` 的模式一致。

### 5. `app/tools/knowledge_tool.py` — 自动触发 + 结果合并

**改了什么**：

1. 新增 `_get_top_score()` — 获取文档列表最高分（优先 `_rerank_score`，其次 `_hybrid_score`）
2. 新增自动触发逻辑 — 重排后检查最高分 < 阈值时，调用 `web_search_service.search()`

## 五、配置说明

### 最小配置（开启网络检索）

在 `.env` 中设置：
```
WEB_SEARCH_ENABLED=True
TAVILY_API_KEY=tvly-你的API密钥
```

### 完整配置项

| 配置 Key | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `WEB_SEARCH_ENABLED` | bool | False | 总开关，需配合 TAVILY_API_KEY |
| `TAVILY_API_KEY` | str | "" | Tavily API 密钥 |
| `WEB_SEARCH_MAX_RESULTS` | int | 5 | 每次搜索返回最大结果数 |
| `WEB_SEARCH_SEARCH_DEPTH` | str | "basic" | basic（快）/ advanced（全面但慢） |
| `WEB_SEARCH_AUTO_TRIGGER_ENABLED` | bool | True | 是否启用自动触发 |
| `WEB_SEARCH_AUTO_TRIGGER_THRESHOLD` | float | 0.3 | 最高分低于此值时自动触发 |
| `WEB_SEARCH_SUMMARIZATION_ENABLED` | bool | True | 是否压缩网络结果 |
| `WEB_SEARCH_SUMMARIZATION_MODEL` | str | "qwen-turbo" | 摘要压缩模型 |
| `WEB_SEARCH_MAX_CONTENT_LENGTH` | int | 2000 | 每条结果截断前最大字符 |
| `WEB_SEARCH_SAFETY_ENABLED` | bool | True | 是否启用安全过滤 |
| `WEB_SEARCH_SAFETY_KEYWORD_BLACKLIST` | str | "" | 关键词黑名单（逗号分隔） |
| `WEB_SEARCH_SAFETY_BLOCKED_DOMAINS` | str | "" | 屏蔽域名（逗号分隔） |
| `WEB_SEARCH_LLM_REVIEW_ENABLED` | bool | True | 是否启用 LLM 安全审核 |
| `WEB_SEARCH_LLM_REVIEW_MODEL` | str | "qwen-turbo" | LLM 审核模型 |

## 六、安全机制详解

### 双层过滤流程

```
网络检索结果
    │
    ▼
[第一层：规则引擎] ← 同步，毫秒级
    ├── 关键词黑名单 → 命中则整篇丢弃
    ├── 屏蔽域名检测 → 命中则整篇丢弃
    └── XSS/注入清洗 → 去除危险模式，保留文档
    │
    ▼
[第二层：LLM 审核] ← 异步，秒级（可选）
    ├── qwen-turbo 判定每篇文档 is_safe
    ├── 不安全文档被过滤
    └── 失败时 fail-open（放行）
    │
    ▼
安全文档 → 进入摘要压缩
```

### 安全配置示例

```
# 屏蔽赌博和色情关键词
WEB_SEARCH_SAFETY_KEYWORD_BLACKLIST=赌博,色情,诈骗,毒品

# 屏蔽已知恶意域名
WEB_SEARCH_SAFETY_BLOCKED_DOMAINS=malicious.com,phishing.net
```

## 七、验证步骤

1. **配置**：设置 `.env` 中 `WEB_SEARCH_ENABLED=True` + `TAVILY_API_KEY`
2. **启动**：运行应用，确认无报错
3. **自动触发**：问一个知识库无答案的问题，检查日志是否显示"自动触发网络搜索"
4. **手动触发**：点击前端"网络搜索"开关 ON，提问，确认 Agent 同时使用知识库和网络搜索
5. **安全过滤**：搜索可能返回敏感内容的词，确认被过滤
6. **降级测试**：设置无效 API Key，确认对话仍正常（仅知识库结果）
7. **关闭测试**：设置 `WEB_SEARCH_ENABLED=False`，确认所有功能不受影响
