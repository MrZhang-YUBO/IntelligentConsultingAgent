### 🔧 1. 新增脚本

| 脚本文件 | 阶段 | 功能说明 |
|---------|------|---------|
| `llm/prepare_lawzhidao_sft.py` | 数据准备 | 从 CSV 筛选 `is_best==1` 高质量问答，输出 seed(200) + SFT 全量 JSON |
| `llm/generate_self_instruct_qwen_api.py` | 数据准备 | Self-Instruct 扩充：用 seed 做 few-shot，调用 Qwen2.5-32B API 生成新问答 |
| `llm/build_sft_dataset.py` | 数据准备 | 三源融合（alpaca + lawzhidao + self-instruct）→ 过滤 → 9:1 切分 train/val |
| `llm/generate_dpo_candidates_qwen_api.py` | DPO Step-1 | 对问题生成 A/B 双候选（temp=0.2 vs temp=0.9，制造偏好差异） |
| `llm/judge_dpo_pairs_qwen_api.py` | DPO Step-2 | 用 Qwen API 作为裁判，标注 `chosen`/`rejected` 供 DPO 训练 |
| `llm/infer_lora.py` | 推断 | SFT LoRA 推断脚本，支持 base + LoRA，输出 `{query, answer, LLM_answer}` |
| `llm/infer_dpo_lora.py` | 推断 | DPO LoRA 推断脚本，支持两种模式：<br>• `MODE=dpo`：base + DPO LoRA<br>• `MODE=sft_dpo`：base + SFT LoRA + DPO LoRA |

### 🎯 2. SFT 训练参数

**训练命令**（`dataset/sft/训练参数.txt`）：

```bash
llamafactory-cli train \
  --stage sft \
  --model_name_or_path /mnt/systemDisk/model_cache/Qwen/Qwen2.5-7B-Instruct \
  --finetuning_type lora \
  --lora_target all \
  --lora_rank 8 \
  --lora_alpha 16 \
  --lora_dropout 0 \
  --template qwen \
  --dataset sft_law \
  --dataset_dir data \
  --cutoff_len 768 \
  --num_train_epochs 1.0 \
  --max_samples 100000 \
  --per_device_train_batch_size 4 \
  --gradient_accumulation_steps 4 \
  --learning_rate 1e-5 \
  --lr_scheduler_type cosine \
  --max_grad_norm 1.0 \
  --warmup_steps 0 \
  --bf16 True \
  --flash_attn auto \
  --optim adamw_torch \
  --logging_steps 10 \
  --save_steps 500 \
  --output_dir saves/Qwen2.5-7B-Instruct/lora/train_fast_768_ep1 \
  --preprocessing_num_workers 16 \
  --packing False \
  --enable_thinking True \
  --report_to none \
  --plot_loss True \
  --trust_remote_code True \
  --ddp_timeout 180000000 \
  --include_num_input_tokens_seen True
```

**SFT 参数要点**：

| 参数 | 值 | 说明 |
|-----|----|------|
| `cutoff_len` | 768 | 99% 数据 < 721 tokens，768 足够，大幅省显存/提速 |
| `num_train_epochs` | 1.0 | 单 epoch，避免过拟合 |
| `per_device_train_batch_size` | 4 | + `gradient_accumulation_steps=4` → **有效 batch=16** |
| `learning_rate` | 1e-5 | LoRA 常用学习率 |
| `lora_target` | all | 对全部线性层做 LoRA 微调，适配性最好 |
| `lora_rank / alpha` | 8 / 16 | alpha = 2×rank，经典配置 |
| `bf16` | True | bf16 训练，精度/显存/速度平衡 |
| `flash_attn` | auto | 自动启用 Flash Attention，加速显存友好 |
| `output_dir` | `saves/.../train_fast_768_ep1` | 产物：adapter_config.json + adapter_model.safetensors |

### 🎯 3. DPO 训练参数

**训练命令**（`dataset/dpo/dpo参数.txt`）：

```bash
CUDA_VISIBLE_DEVICES=0 llamafactory-cli train \
  --stage dpo \
  --do_train \
  --model_name_or_path /mnt/systemDisk/model_cache/Qwen/Qwen2.5-7B-Instruct \
  --dataset dpo_law \
  --dataset_dir data \
  --template qwen \
  --finetuning_type lora \
  --lora_target all \
  --output_dir outputs/qwen2.5-7b_lora_DPO_dpo_law_len512_bs2_ga4_beta0.1 \
  --overwrite_output_dir \
  --cutoff_len 512 \
  --per_device_train_batch_size 2 \
  --gradient_accumulation_steps 4 \
  --learning_rate 5e-6 \
  --num_train_epochs 1.0 \
  --lr_scheduler_type cosine \
  --warmup_ratio 0.05 \
  --fp16 \
  --logging_steps 10 \
  --save_steps 200 \
  --save_total_limit 2
```

**DPO 参数要点**：

| 参数 | 值 | 说明 |
|-----|----|------|
| `stage` | dpo | Direct Preference Optimization，偏好优化 |
| `cutoff_len` | 512 | 比 SFT 更短，DPO 输入是 prompt + chosen + rejected，省显存 |
| `per_device_train_batch_size` | 2 | + `gradient_accumulation_steps=4` → **有效 batch=8** |
| `learning_rate` | 5e-6 | DPO 通常用比 SFT 更小的 LR（约 1/2） |
| `beta` | 0.1 | DPO loss 的 KL 惩罚系数，默认=0.1，越小越偏离参考模型 |
| `warmup_ratio` | 0.05 | 5% 步数 warmup |
| `fp16` | True | fp16 训练（SFT 用 bf16，可统一为 bf16） |
| `save_steps` | 200 | 每 200 步保存一次 |
| `save_total_limit` | 2 | 只保留最近 2 个 checkpoint，省磁盘 |
| `output_dir` | `outputs/.../DPO_dpo_law_len512_bs2_ga4_beta0.1` | DPO LoRA 权重产物 |

### 📊 4. 数据构建流程

#### 4.1 SFT 数据流

```
lawzhidao_filter.csv
    └─► prepare_lawzhidao_sft.py
        ├─► lawzhidao_best_sample200.json  (seed for self-instruct)
        └─► lawzhidao_best_sft_all.json    (SFT 全量)
    └─► generate_self_instruct_qwen_api.py (Qwen2.5-32B API)
        └─► self_instruct_gen_qwen32b.jsonl (≈500 条扩充)
    └─► build_sft_dataset.py
        ├─► 合并 3 来源：alpaca + lawzhidao + self-instruct
        ├─► 过滤：空 instruction/short output (<15 tokens)
        ├─► random shuffle
        └─► 9:1 切分
            ├─► sft/train.jsonl
            └─► sft/val.jsonl
```

#### 4.2 DPO 数据流

```
intent_2k.json（法律问题集，类型=法律类）
    └─► generate_dpo_candidates_qwen_api.py (Qwen2.5-32B API)
        │  temp_A=0.2（保守答案）
        │  temp_B=0.9（发散答案）
        └─► dpo/raw/dpo_candidates_ab_qwen32b.jsonl
               {instruction, cand_a, cand_b}
    └─► judge_dpo_pairs_qwen_api.py (Qwen API 裁判)
        │  判断 A/B 优劣 → {better: "A"|"B"}
        └─► dpo/dpo_pairs_qwen32b.jsonl
               {instruction, input, chosen, rejected}
```

### 🚀 5. 推断/部署

### 🔄 6. 配置特性

| 特性 | 说明 |
|-----|------|
| **Prompt 外部化** | 所有 API 调用 prompt 不硬编码，从 `experiments/<exp>/prompts/` 读取 |
| **环境变量配置** | 模型路径、数据路径、生成参数全部可用环境变量覆盖 |
| **断点续跑** | self-instruct / dpo-judge 脚本检查已有 output 行数，支持中断后继续 |
| **指数退避重试** | 所有 DashScope API 调用失败自动指数退避重试（默认 4-5 次） |
| **LoRA 动态加载** | `infer_*.py` 用 PEFT `PeftModel.from_pretrained`，不做权重合并 |
| **输出命名约定** | output 目录自带参数编码（如 `len512_bs2_ga4_beta0.1`），便于实验追溯 |

### ⚠️ 7. 已知问题 / 后续优化

- [ ] **精度类型不统一**：SFT 用 `--bf16`，DPO 用 `--fp16`。建议统一为 `--bf16`（若 GPU 支持，如 A10/A100）以保持训练一致性。
- [ ] **DPO beta 参数**：当前 `beta=0.1`（通过 output_dir 命名推断），但命令行未显式传入。建议在命令中加 `--dpo_beta 0.1` 让配置更明确。
- [ ] **save_steps 不一致**：SFT=500，DPO=200。建议根据实际 loss 曲线统一或按需调整。
- [ ] **warmup 配置差异**：SFT 用 `--warmup_steps 0`，DPO 用 `--warmup_ratio 0.05`。DPO 有 warmup 更合理，建议 SFT 也加上。
- [ ] **DPO 数据量**：当前 A/B 候选数量无明确记录，建议在脚本中打印最终 DPO 样本量统计。
- [ ] **推断评估自动化**：当前 `infer_*.py` 仅生成 `LLM_answer`，缺少自动化指标（如 ROUGE/BLEU/人工评测脚本）。

### 🛠️ 9. 快速复用（下一次实验）

1. **准备新数据**：替换 CSV 或新增数据源 → 跑 `prepare_lawzhidao_sft.py`
2. **扩充数据（可选）**：调 `SELF_INSTRUCT_N_SAMPLES` 跑 `generate_self_instruct_qwen_api.py`
3. **构建 SFT 数据集**：跑 `build_sft_dataset.py`
4. **SFT 训练**：复制 `dataset/sft/训练参数.txt`，按需调 lr/epoch/cutoff_len
5. **生成 DPO 候选**：跑 `generate_dpo_candidates_qwen_api.py`（调 `DPO_N` 控制样本量）
6. **裁判标注**：跑 `judge_dpo_pairs_qwen_api.py`
7. **DPO 训练**：复制 `dataset/dpo/dpo参数.txt`，按需调 `beta`/`cutoff_len`
8. **推断对比**：分别跑 `infer_lora.py` / `infer_dpo_lora.py`（MODE=sft_dpo）对比基线 → SFT → SFT+DPO 的效果差异
